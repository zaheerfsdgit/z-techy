In Spring Boot, 'Boot' refers to the concept of rapidly initiating a spring application

Spring Boot makes building Spring applications easier and faster.

Core Principles of Spring Boot 


-- Opinionated Defaults 

    It means, Spring Boot Internally handled the build tools like Maven or Gradle.

    and while we create the project we can specify required dependencies as per 
    our project requirement, then spring boot internally add all depependacies.

    and coming to all basic setups also taken care by spring bootl like
    project structure also spring boot will create internally.

    and It also simplifies the basic configurations by providing "application.properties".

    ** [Appliation.properties file used to provide the configuration in Spring Boot appliation,
    This file replaces the need of spring configuration like mydps-servlet.xml file ]

-- Convention Over Configuration 

    It means, it we follow certain established patterns or conventions, we can achieve a lot
    without needing to explicitly configure everything.

    And it is achieved by Annotataionls, Example We use @RestController for creating RESTful
    API's, @Service for service layers, or organizing our code into specific subpackages to 
    enable automatic Scanning.

-- Auto Configurations 

    It means, setting up components and settings automatically based on project dependencies.
    For Example, Including spring-web configures essestials like the dispatcher servlet and view
    resolver for web apps, eliminating manual setup and letting developers focus on coding
    instead of infrastructure. 

-- Embedded Servers 

    In Spring Boot Server has been embedded internally, no need to configure from out end.
    While using spring framework we face server version conflicts, so that Spring boot
    remove these isssues by embedded the server.

Spring Boot Starters


 -- Understanding Spring Boot Starters 

    Spring Boot Starters are the "Dependancy Discriptors"

    Dependancy -- Expternal libraries or JARs.
    Descriptor -- Configuration specifying JARs and thier videos.

    ** Spring Boot Starters combine all necessary libraries (JAR files) for a 
    particular feature or technology into a single dependancy.

    It means, you will use only one Dependancy but it includes many JAR files. 

    For Example: Spring-Boot-Starter-Web Dependancy contains many jar files like

    -- Web App
    -- web.jar 
    -- web-mvc.jar 
    -- validation.jar 
    -- tomcat.jar 
    -- logging.jar 
    etc

 -- Uses of Spring Boot Starters 

    Simplifies Configuration : Statters automatically include the right libraries and settings 
    we need, so that we write code faster.

    Boost Productivity : Saves time and Productivity by providing sensible defaults and configurations,
    allowing developers to focus on writing business logic rather than managing dependies.

    Ensures Compatability : Ensures that all included depenencies  are compatible with each other and 
    the Spring Boot version being used, reducing the risk of conflicts.
    
 -- Important Spring Boot Starters 

     1. spring-boot-starter

        It is the core starter that includes auto-configruring spport, logging, And YAML processing.

     2. spring-boot-starter-web

        It provides dependencies to build web applications, including Spring MVC, RESTful services, 
        and embedded Tomcat.

     3. spring-boot-starter-data-jpa

         It provides dependencies for Spring Data JPA, Hibernate and Database connection pool.

     4. spring-boot-starter-security

        It inlcudes Spring Security and its dependencies for securing applications.

     5. spring-boot-starter-thymeleaf

        It inegerates the Thymeleaf templating engine with Spring MVC.

     6. spring-boot-starter-test

        It includes testing libraries like JUnit, Hamcrest, and Mockito.

     7. spring-boot-starter-actuator

        It provides production-ready features to help monitor and manager your appliatoin,
        including health checks and metrics.

     8. spring-boot-starter-aop

        It brings in support for aspect-oriented programming using Spring AOP and AspectJ.

All About Spring Boot JDBC Appliation 


    Steps for Spring Boot Jdbc Application..

    By Using Spring Boot JDBC, we can connect our Spring Boot Application 
    to database in a simplified manner.

    How Spring Boot Simplified the things ??

    ** Because Spring Boot automates many configuratoin tasks, allowing us to focus on writing 
    the application logic.

   There are 3 Main steps to perfomr Spring Boot Jdbc: 

    1. Provide "jdbc" and "mysql" starter dependencies
        spring-boot-starter-jdbc 
        spring-boot-starter-mysql 

    2. Provide database properties in application.properties or application.yml files

    application.properties

        spring.datasource.url = jdbc:mysql://localhost:3306/mydb
        spring.datasource.driver-class-name = com.mysql.cj.jdbc.Driver 
        spring.datasource.username = root 
        spring.datasource.password = root


    application.yml

    spring:
        datasource:
            url: jdbc:mysql://localhost:3306/your_database_name
            username: your_username
            password: your_password
            driver-class-name: com.mysql.cj.jdbc.Driver
        jpa:
            hibernate:
                ddl-auto: update
            show-sql: true

     Spring Boot auto-configures the DataSource 
    Based on the properties provided in 
    application.properties file. 

    3. Use "JdbcTemplate" class for database operations.

Important Points About Spring Boot JDBC 


** Spring Boot JDBC is built upon Spring JDBC Module.

** Spring Boot auto-configures the DataSource, JdbcTemplate / NamedParameter JDBC Template 
    beans, reducing manual configurations.

    DataSource :     Interface for configuring and managing database connection.

JdbcTemplate   :     Class providing simplified JDBC operations and query exection.

NamedParameter      Class enabling JDBC operations with named parameters
 JdbcTemplate  :     for better readability.

if(i>0)
{
    System.out.println(" true Stauts captured");
    status = true;      
}
else
{
    System.out.println("false Stauts captured");
    status = false;    
}

}
catch(Exception e)
{
    e.printStackTrace();
}
return status;
}

}
