Spring Introduction 


Spring is an open source java frame work

It is used intostand alone and enterprise app's

It is released in 2004 and developed by "Rod Johnson"

Java spring configuration done by 3 ways: 

    1. xml based

    2. Java based

Spring Container [Core Component] 


Spring Container is consider as the heart of the Spirng.

like wise JVM is the heart of Java.

Spring Container Responsiblities: 

    1. Manage bean Objects

    2. Manage bean life cycle

    3. DI / Dependency Injection

    4. AOP / Aspect Ojbect Programming

    5. Transaction Management

    6. Internationalize (I18N)

Spring Container are two types:

    1. Bean Factory (Old)
    2. Application Context (new)

    ** (Application Context is basically a Interface)

     Note:  Spring Container is invoked / start by Application Context,
     It means Spring Container Responsibilities executed by Application Context.

Dependency Injection 


Dependency Injectionit is a Design Pattern.

And It is used in the spring framework to achieve Inversion of Control (Ioc).

Its main task is to "inject" the "dependency" means inject one object into 
another object.

It is used to achieve loose coupling in java.

Basically core java do tight  coupling. 

Dependancy Injection achieved with XML and Java based Configuration,
with Setter Injection and Constructor Injection.

We can find both processes xml based and java based annotations at below

Auto Wired 


Auto Wired is the process to convert the Manual Dependancy Injection to 
Auto Dependancy Injection.

The @Autowired annotation in Spring is used for automatic dependancy injection.

Autowired is used to achieve "DI Automatically"

Autowired annotatoin tells to Spring grab the right peace (Bean) and plug it
respective bean to work automatic Dependancy Injection.

In short, it simplifies your code by letting Spring handle connections between
Objects to executes proper Dependancy Injection.

Basically, Dependancy Injection perform manually without Autowired.

With Autowiring we achive Automatic Dependancy Injection.

Autowiring can be achieved by 2 Ways: 

  -- Annotation Based Autowiring -> @Autowired
  -- XML Based Autowiring  -> "autowire" attribute (modes)


Auto Wiring => Auto + Wiring

"Auto" Means Automatically manages the connection between Objects.

"Wiring" Means Linking those objects together to fulfill dependencies.
